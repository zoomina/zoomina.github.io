# 코틀린 소개
----
- 2016년 2월 1.0 정식 버전 출시
- Google I/O 2017에서 안드로이드의 공식 지원 언어로 채택
- 간결한 문법, 호환성, 높은 안정성을 토대로 한 높은 생산성
- JAVA와 100% 호환  
  : JVM 위에서 실행

- intelliJ, eclipse, Visual Studio, Android Studio 등에서 코틀린을 지원하고 있으니 각자 목적에 맞는 툴을 사용하면 된다.  
  - Android Studio는 앱개발을 목표로 하기 때문에 코틀린을 배우는 목적으로는 적합하지 않다. intelliJ가 가장 유사하기 때문에 입문용으로 추천이 많은 것 같다.
  - Visual Studio에서도 Kotlin을 지원하고 있지만, 기능을 이용하는데 제한이 생겨 크게 추천하지는 않는 모양이다. 
  - 본인은 앱개발을 목표로 하고 있지만, 현재 진행중인 스터디 전원이 입문단계이고 eclipse가 설치되어 있어 이를 이용하고 추후 Android Studio로 넘어가기로 했다.

# 코틀린의 특징
----
: 앞서 코틀린의 장점으로 소개한 간결한 문법, 호환성, 높은 안정성에 대해 간략하게 살펴볼 예정이다.  
필자는 입문자에 불과하기 때문에 이러한 장점들을 크게 체감하고 있지 못해, 표현과 정보에 부정확한 부분이 있을 수 있다.

1. 간결한 문법  
  : 코틀린은 일단 JVM 위에서 돌아가기 때문에 JAVA와 거의 같다고 볼 수 있다. 하지만 문법은 그에 비해 간결하기 때문에 코드 작성을 40%까지도 단축시킬 수 있다고 한다. 여기서는 가장 기본적인 문법을 비교해보려고 한다.

  - 문장 끝에 세미콜론(;)을 넣지 않아도 됨
  - new 키워드를 쓰지 않고 객체를 생성
  - 타입 추론을 지원하므로 일반적인 경우 타입을 적지 않아도 됨
  ```JAVA
  // JAVA
// 타입을 명시해주어야 한다.
  String name = "mina";
  // 객체 생성시 new 키워드를 사용한다.
  Person person = new Person(name);
  ```
  ```JAVA
  // Kotlin
  // 타입 추론을 통해 String 타입으로 지정해준다.
  val name = "mina"
  //객체 생성시 new 키워드를 사용하지 않는다.
  val person = Person(name)
  ```

2. 호환성  
  : 코틀린은 JAVA, Java Script, Android, 서버측 웹 애플리케이션 등 다양한 분야에 호환된다. 기존의 언어에 비해 간결한 문법으로 다양한 분야에 호환이 될 수 있다는 점은 코틀린의 또 하나의 강점이라고 볼 수 있다.

3. 높은 안정성
  - Null 안정성  
  : NullPointerError에 대한 대응책을 여럿 제공하고 있기 때문에 Null 안정성이 높다.
  
  - 가변/불변 구분  
  : value와 variable을 구분하여 선언하기 때문에 변수와 상수에 대한 안정성이 높다.  
    - val : JAVA의 final 변수와 같이 변경할 수 없는 변수
    - var : 할당된 값을 바꿀 수 있는 변수
  
  ```Java
  val data1:Int = 10
  val data2 = 20
  var data3 = 30
  
  println("data1:" + data1)
  println("data2:" + data2)
  println("data3:" + data3)
  data2 = 40  // value는 변하지 않는다.
  data3 = 40
  println("data3:" + data3)
  ```
  출력 결과
  ```
  >>> data1: 10
  >>> data2: 20
  >>> data3: 30
  >>> data3: 40
  ```
