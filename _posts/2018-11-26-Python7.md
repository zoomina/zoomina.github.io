Python은 인터프리터 방식의 언어로 절차지향, 객체지향, 함수형 프로그래밍 등 다양한 형태의 프로그래밍을 구사할 수 있는 언어입니다.  
그리고 객체지향 프로그래밍에서 중요한 개념 중 한 가지인 클래스를 이번 포스팅을 통해 살펴볼 예정입니다.  

<br>

## 클래스?
----

이전 포스팅에서 다룬 함수가 하나의 기능 단위였다면, 클래스는 그보다 많은 것을 포함합니다.  
클래스를 한 마디로 정리하자면 '프로그램이라고 부를 수 있는 최소 단위' 라고 할 수 있을 것입니다.  
물론 이는 설명의 편의를 위한 표현일 뿐, 정확한 표현이 아니며 언어와 프로그래밍 방식에 따라 달라질 수 있습니다.  
<br>
클래스는 멤버 변수(field)와 메서드(method)로 구성되어 있습니다.  
메서드는 클래스 내부에 정의된 함수를 의미합니다.  

<br>

## 클래스의 구조
----

### 1. 클래스의 정의 방법

Python에서 클래스를 정의할 때 사용하는 예약어는 `class` 입니다.  

```python
class <클래스 이름>(상속할 클래스):
    <field>
    <method>
```

함수를 정의할 때와 비슷하지만 내부에 들어가는 것이 더 늘었습니다.  
내부에 하나의 작은 프로그램 단위를 작성한다는 느낌으로 접근하는 게 좋을 것 같습니다.  

이때, 클래스 옆의 괄호 안에는 상속받을 클래스를 입력합니다.  
만약 상속할 클래스가 없다면 아래 이어지는 예시들처럼 최상위 클래스인 `object`를 상속합니다.  
상속에 대한 개념은 복잡할 수 있고, Python에서는 제가 잘 사용하지 않기 때문에 정확하지 않을 수 있어 이번 포스팅에서는 다루지 않을 예정입니다.  

### 2. 클래스의 메서드

클래스의 메서드는 일반적으로 함수를 정의하는 것과 비슷하게 작성합니다.  
차이가 있다면 매개변수로 클래스 자신(self)을 넘겨주어야 합니다.  

```python
class MyClass(object):
    def fun(self):
      <함수내용>
```

다른 매개변수를 작성한다면 self 뒤에 차례로 작성하면 됩니다.  


<br>

클래스의 메서드를 바깥에서 사용할 때는 평소 모듈을 사용할 때와 비슷하게 사용합니다.  
위에 정의한 fun이라는 메서드를 사용한다면 `MyClass.fun()` 이라고 사용합니다.  

### 3. 생성자

생성자는 매개변수를 받아 클래스 내부에서 사용할 수 있는 형태로 변환해주는 역할을 합니다.  
생성자는 객체가 호출되는 순간 자동으로 호출됩니다.  

```python
class Student(object):
  def __init__(self, name, grade):
    self.name = name
    self.grade = grade
    print('init ing...', name, grade)

person = Student('mina', '2')
```

**실행 결과**

```
init ing... mina 2
```

위의 예시에서 볼 수 있듯이 생성자는 `__init__`이라는 특별 메서드를 사용합니다.  
이때 주의할 점은 `_`를 두 번 입력해주셔야 한다는 점입니다!!  
앞 뒤로 언더바(`_`)를 두 번 입력하는 메서드들을 특별 메서드라고 합니다.  
지금까지는 `__init__`이 처음이지만 앞으로 Python을 사용하면서 종종 마주칠 수 있습니다.
