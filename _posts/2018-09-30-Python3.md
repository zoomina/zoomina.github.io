## 컬렉션 자료형
----

컬렉션 자료형은 리스트, 튜플, 셋, 딕셔너리가 있습니다.
얼핏 비슷한 것 같지만 조금 다른 네 가지 자료형에 대해 소개하겠습니다.

#### 1. 리스트[ ]

리스트는 여러 개의 자료를 나열하는 자료형입니다.
마치 C나 Java에서의 배열과 같은 존재입니다.
리스트는 수정과 삽입이 가능하기 때문에 일반적으로 여러 개의 자료를 나열할 때는 리스트를 사용하게 됩니다.

리스트를 만드는 방법은 간단합니다.

```python
alist = [1, 2, 3, 4, 5]
```

리스트는 소제목 옆에 표기된 대괄호를 이용해 선언합니다.  
`name = [data1, data2, data3, ...]`

<br>

앞서 리스트는 수정과 삽입이 가능하다고 언급했습니다.
이번에는 리스트의 수정/삽입에 대해 살펴보도록 하겠습니다.

리스트도 문자열처럼 인덱스를 갖습니다.
그렇기에 인덱스를 이용한 호출과 슬라이싱이 가능합니다.
그리고 문자열처럼 `+`, `*` 연산을 사용할 수 있습니다.
예제 코드를 통해 확인하도록 하겠습니다.

```python
alist = ['mon', 'tue', 'wed', 'thu', 5, 6, 7]
blist = [1, 2, 3, 4, 'fri', 'sat', 'sun']

day = alist[:4] + blist[4:]
num = blist[:4] + alist[4:]

print(day)
print(num)

>>> 출력결과
['mon', 'tue', 'wed', 'thu', 'fri', 'sat', 'sun']
[1, 2, 3, 4, 5, 6, 7]
```

<br>

그리고 `append()` 함수를 이용하면 리스트에 요소를 추가하는 것이 가능합니다.

```python
num = [1, 2, 3, 4, 5]
num.append(6)
print(num)

>>> 출력결과
[1, 2, 3, 4, 5, 6]
```

물론 `+` 연산을 이용해 추가할 수도 있지만, 반복적으로 여러 개의 요소를 추가할 때는 이 함수를 유용하게 사용하게 될 것입니다.

<br>


#### 2. 튜플( )

튜플도 리스트와 마찬가지로 여러 개의 자료를 나열하는 자료형입니다.  
하지만 다른 점이 있다면 튜플은 수정과 삽입이 불가능합니다.  

튜플을 선언하는 방법 또한 리스트와 같습니다.

```python
atup = (1, 2, 3)
```

<br>

튜플은 주로 함수를 통해 자료를 주고 받을 때, 한 번에 초기화 시킬 때 등 **자료의 틀을 잡아주는 용도**로 사용합니다.  

```python
(x, y) = (3, 4)

print("x축:", x, "y축:", y)

>>> 출력결과
x축: 3 y축: 4
```

<br>

#### 3. 셋{ }

셋은 앞서 살펴본 리스트, 튜플과는 조금 다른 느낌의 자료형입니다.  
셋은 **중복되지 않고**, **순서가 없는** 자료형입니다.  
따라서 중복되는 자료를 넣으면 하나만 저장이 되고, 출력할 때에도 순서가 바뀌어 출력되기도 합니다.  
순서가 없기 때문에 인덱스를 사용할 수 없습니다.  

```python
aset = {1, 1, 1, 2, 3, 3, 4, 4}
print(aset)

>>> 출력결과
{1, 2, 3, 4}
```

하지만 값의 추가와 삭제는 가능합니다.  

```python
aset = {1, 2, 3, 4}
# 5를 추가
aset.add(5)
# 4를 제거
aset.remove(4)

print(aset)
>>> {1, 2, 3, 5}

bset = {1}
# 한 번에 여러 데이터를 추가
bset.update({2, 3, 4})
print(bset)
>>>{1, 2, 3, 4}

# 모든 데이터를 제거
bset.clear()
print(bset)
>>> set()
```

<br>

그리고 셋은 수학에서의 집합에 대한 연산이 가능합니다.  

연산자 | 메서드 | 기능
:---: | :---: | :---:
& | union() | 교집합  
&brvbar; | intersection() | 합집합  
- | difference() | 차집합  

```python
aset = {1, 2, 3, 4}
bset = {1, 3, 5, 7}
print(aset & bset)
>>> {1, 3}
print(aset | bset)
>>> {1, 2, 3, 4, 5, 7}
print(aset - bset)
>>> {2, 4}
```

<br>

#### 4. 딕셔너리{ : }

딕셔너리는 셋과 같은 모양의 괄호를 사용합니다.  
기능 역시 셋에서 확장된 개념으로 생각하시면 이해가 편합니다. (제가 구분하는 방법일 뿐 정확한 정의는 아닙니다.)  
- 딕셔너리는 key와 value를 갖습니다.  
  - key : value
- key는 중복될 수 없습니다.  
- 딕셔너리도 셋과 마찬가지로 순서가 없는 자료형입니다.

```python
adict = {'a':1, 'b':3, 'c':3}
bdict = {1:4, 1:3, 3:2}

print(adict)
>>> {'a': 1, 'b': 3, 'c': 3}
print(bdict)
>>> {1: 3, 3: 2}
```
위와 같이 value는 중복될 수 있지만, key는 중복될 수 없습니다.  
중복되는 key값이 들어오면 뒤에 들어온 value로 수정됩니다.  

<br>

예제코드를 이용해 딕셔너리의 수정, 추가, 제거를 살펴보겠습니다.  

```python
adict = {'a':1, 'b':3, 'c':3}

# a라는 key의 value를 수정
adict['a'] = 3
# d라는 key를 추가
adict['d'] = 1
# c라는 key를 제거
del adict['c']

print(adict)
>>> {'a': 3, 'b': 3, 'd': 1}
```

<br>
<br>


(+) 추가로 말씀드리자면..  

저는 현재 자연어처리를 공부하는 중이라 딕셔너리를 주로 단어의 개수를 세는 용도로 사용하고 있습니다.  
앞서 셋에서 확장된 개념으로 이해하고 있다고 말씀드렸는데요, 그 이유는 다음과 같습니다.  

- 셋은 중복되는 데이터를 모두 지운다. 결과적으로 데이터의 빈도를 알 수는 없다. 딕셔너리를 사용하면 데이터의 빈도수를 value로 표현할 수 있다.  

- 셋은 순서가 없기 때문에 데이터에 접근하려면 직접 값을 입력해야한다. 즉, 모든 데이터를 알아야 수정이 가능하다. 딕셔너리는 key값을 통해 접근하기 때문에 모든 value를 알 필요는 없다.
