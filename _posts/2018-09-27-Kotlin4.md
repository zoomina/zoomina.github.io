## 함수 선언(fun)
----

- 함수를 선언할 때는 `fun`이라는 예약어를 사용합니다.
```
fun sum(a:Int, b:Int):Int {
      return a + b
}
```

- 함수의 매개변수는 `var`입니다.

- 반환값이 없을 때는 `Unit`으로 입력하고, 이는 생략할 수 있습니다.
```
fun show(name:String): Unit {
      println("Hello" + name)
}
```
```
fun show(name:String) {
      println("Hello" + name)
}
```

- 함수 내에 함수를 선언할 수 있습니다.

- 단일 구문은 중괄호`{}`를 생략하고 등호`=`를 사용해 정의할 수 있습니다.(단일 표현 함수)
```
fun show(name:String) = println("Hello" + name)
```

- 단일 표현 함수는 반환 타입을 생략할 수 있습니다.
```
fun sum(a:Int, b:Int) = a + b
```

<br>

## 함수 오버로딩(overloading)
----

매개변수는 다르지만 이름은 같은 함수를 여러 개 정의하는 기법입니다.  
이는 코틀린 외의 다른 언어에서도 많이 지원하는 기능입니다.

```
fun show(a:String) {
      println(a)
}
fun show(a:Int) {
      println(a)
}
fun show(a:Int, b:String) {
      println("$a " + b)
}

fun main(args: Array<String>) {
      show("Hello")
      show(2018)
      show(2018, "Hello")
}

>>> 출력결과
Hello
2018
2018 Hello
```

<br>

## 인수(argument)
----
Java와 마찬가지로 Kotlin역시 기본 인수(default argument)를 지원합니다.  
기본 인수는 호출자가 인수를 명시하지 않아도 알아서 기본값을 적용할 수 있게 해줍니다.  

```
fun hello(name: String? = "mina") {
      println("Hello " + name)
}

fun main(args: Array<String>) {
      hello()
      hello("zooA")
}

>>> 출력결과
Hello mina
Hello zooA
```

<br>
단, 파라미터가 여러 개인 경우 변수를 명확하게 지정해야 합니다.  
```
fun hello(name:String = "mina", no: Int) {
      println("$no Hello " + name)
}

fun main(args: Array<String>) {
      hello(10)   // 오류발생
      hello(no = 10)
      hello("zooA", 10)
}
```

## 중위 표현식(infix)
----
중위 표현식이란, 우리가 연산자를 사용하는 방식입니다.  
사실 컴퓨터는 후위 표현식(1 2 3 + -)을 더 선호하지만, 우리는 중위 표현식(1 + 2 - 3)이 더 익숙합니다.  
Kotlin에서는 함수 호출에 중위 표현식을 사용할 수 있습니다.

- 함수를 중위 표현식으로 이용하려면 infix라는 예약어로 선언해야 합니다.
- infix 사용을 위한 조건은 아래와 같습니다.
  - 클래스의 멤버 함수로 선언하거나 클래스의 확장 함수일 때
  - 매개변수가 하나인 함수일 때

```
// 함수 확장
infix fun Int.myFun(x:Int): Int {
      return x * x
}

class FunClass {
      infix fun infixFun(a:Int) {
          println("infixFun call")
      }
}

fun main(args: Array<String>) {
      val obj = FunClass()
      // 일반적인 표현식
      obj.infixFun(10)
      // 중위 표현식
      obj infixFun 10

      // 중위 표현식
      println(10 myFun 10)
      // 일반적인 표현식
      println(10.myFun(10))
}
```

## 가변 인수(vararg)
----

앞서 살펴본 오버로딩 기능을 이용하면 너무 많은 함수가 선언되어 불편함이 생깁니다.  
이런 경우 가변 인수(variable argument)를 포함한 함수를 선언하여 다음과 같이 쉽게 작성할 수 있습니다.  

```
fun <T> varargsFun(a1:Int, vararg array: T) {
    //..
}
```

- 가변 인수를 포함하는 함수는 vararg라는 예약어를 사용합니다.
- 위의 함수는 첫 번째 인수는 Int, 그 뒤로는 타입과 개수가 자유롭습니다.  
- 가변 인수는 명확한 타입이 정해지지 않았기 때문에 제네릭(`<T>`, `vararg array: T`)을 이용합니다.

## 재귀함수(tailrec)
----
재귀함수는 같은 로직을 반복하는 경우 코드를 보다 쉽게 작성할 수 있는 방법입니다.  
하지만 이는 `StackOverflowError`의 부담감이 있습니다.   

하지만 Kotlin에서는 이러한 부담감을 줄여주는 기능을 제공합니다.  
그 기능이 바로 tailrec 재귀함수 입니다.  
tailrec 재귀함수를 이용할 경우, 우리는 재귀함수를 작성하지만 Java로 변환되는 과정에서 반복문으로 바뀌게 됩니다.  
<br>

tailrec 재귀함수를 이용하는 방법은 어렵지 않습니다.
```
tailrec fun tailrecPrint(no:Int = 1, count: Int = 1) {
    println("tailrecPrint..$no")
    return if(no == count) return else tailrecPrint(no -1, count)
}

fun main(args: Array<String>) {
  tailrecPrint(3)
}

>>> 출력 결과
tailrecPrint..3
tailrecPrint..2
tailrecPrint..1
```
- 함수 선언부에 `tailrec`이라는 예약어를 추가합니다.
- 재귀 구문은 함수의 맨 마지막 작업으로 작성합니다.
