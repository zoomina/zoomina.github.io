## 변수 선언하기
----

코틀린에서 변수를 선언하는 방법은 다음과 같습니다.

```Java
[var/val] 변수명: Type = data

// ex
var num: Int = 10
val str: String = "Hello, Kotlin"
```

기존의 언어들과의 차이점이 보이시나요?  
<br>
Kotlin은 Swift, Scalar 등의 최신 언어들처럼 **Type보다 변수명을 먼저 입력**받습니다.  
물론 앞선 포스트에서 언급한 것 처럼 Kotlin은 타입추론을 지원하기 때문에 타입을 반드시 입력하지 않아도 됩니다.  
추가로, 코틀린의 변수는 자동으로 초기화가 되지 않으므로 반드시 **선언과 동시에 초기화**를 시켜주셔야 합니다.  

<br>

그렇다면 `var`과 `val`의 차이는 무엇일까요?  

```Java
var num1 = 10
val str = "Hello, Kotlin"

// 다음은 적합하지 않습니다.
var num2
num2 = 20
```

바로 `var`은 가변 변수(variable)이고, `val`은 불변 변수(value)라는 점 입니다.  
즉, `var`은 일반적인 변수처럼 **수정이 가능**하지만, `val`은 상수처럼 **읽기 전용** 변수입니다.  
`val`을 온전히 상수로 부르지 않는 이유는 Kotlin의 변수는 Java의 변수와는 다른 개념이기 때문입니다.  
이는 뒤에서 다시 한 번 다루도록 하겠습니다.

## nullable 변수
----

앞서 Kotlin에서의 변수 선언 방법에 대해 살펴보았습니다.  
하지만 프로그래밍을 하다보면 분명 선언과 동시에 초기화를 할 수 없는 경우가 발생합니다.  
이런 경우를 위해 Kotlin에서는 nullable 타입의 변수를 지원하고 있습니다.  

```Java
var 변수명: Type? = null

// 다음은 적합하지 않습니다.
val 변수명: Type? = null
var 변수명: Type = null
```

Kotlin에서는 변수 타입 뒤에 물음표(?)를 붙여 nullable 타입을 사용할 수 있습니다.  
이를 통해 변수에서의 null 안정성을 보장합니다.  
<br>
여기서 `var` 타입을 사용하는 이유는 이후 **변수에 새로운 값을 할당**해야하기 때문입니다.  

<br>

## const 상수 선언
----

초반에 `val`은 온전한 의미의 상수라고는 할 수 없다고 했습니다.  
이번에는 Kotlin에서 상수를 선언하는 방법에 대해 살펴보도록 하겠습니다.  

```Java
const val 변수명: Type = data

// 다음은 적합하지 않습니다.
const var 변수명: Type = data
```

Kotlin에서 상수를 선언할 때는 `const` 키워드를 사용합니다.  
<br>
상수를 선언할 때 반드시 `val`을 사용하는 이유는 nullable 변수와 같습니다.  
상수는 변경할 수 없는 데이터이므로 읽기 전용 변수인 `val`을 사용해 선언합니다.

## Kotlin의 변수 프로퍼티(Property)
----

Koltin의 변수는 기존의 변수와는 다른 개념이라고 앞서 말씀드렸습니다.  
이번엔 그 개념에 대해 간단하게 살펴보도록 하겠습니다.  
<br>
Java는 사실 모든 데이터가 객체는 아니었습니다.  
변수는 기본 데이터 타입을 이용하고 있었습니다.  
이 때문에 Java에서는 캡슐화와 은닉화를 위해 변수를 `private`으로 선언하고 getter/setter를 따로 선언해주어야 하는 불편함이 있었습니다.  
<br>
Java의 한계를 보완해 등장한 Kotlin의 특징답게 이러한 한계를 극복한 프로퍼티를 기본 변수 타입으로 사용하고 있습니다.
프로퍼티는 변수값을 저장하는 **field**와 접근자(accessor)인 **getter/setter**로 이루어진 객체입니다.  
즉, Kotlin은 모든 데이터 타입이 객체로 이루어져 있습니다.  

```Java
// 프로퍼티의 구조
var propertyName[: PropertyType][=property_initializer]
[getter]
[setter]
```

위의 프로퍼티의 구조를 보면 알 수 있듯이 프로퍼티는 변수값인 field에 직접 접근하는 것이 아니고 accessor를 통해 접근하여 캡슐화와 은닉화가 이루어진 형태입니다.  
<br>
Kotlin에서의 변수 개념을 이해하기 위해 프로퍼티의 개념에 대해 간단하게 언급하였습니다.  
하지만 이는 좀 더 다룰 내용이 많기 때문에 이후 별도의 포스팅을 마련할 예정입니다.
