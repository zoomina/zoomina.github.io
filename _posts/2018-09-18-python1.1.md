이번 포스팅에서는 지난 포스팅을 바탕으로 진행한 수업에서 나온 질문을 바탕으로 몇 가지 주의사항에 대해 포스팅 할 예정입니다.

### 1. input()은 str타입으로 입력받습니다.  

지난 수업에서 태어난 년도를 입력받아 나이를 계산하는 예제를 제시했는데 많은 분들이 다음과 같은 실수를 했습니다.

```python
birth = input("태어난 년도:")
print("나이:", 2019-birth)
```

이렇게 입력을 받으면 다음과 같이 int형과 str타입 사이에서 -연산을 할 수 없다는 오류가 뜨게 됩니다.

```
Traceback (most recent call last):
  File "<pyshell#2>", line 1, in <module>
    print("나이:", 2019-birth)
TypeError: unsupported operand type(s) for -: 'int' and 'str'
```

물론 파이썬에서는 변수를 초기화시키면 자동으로 데이터 타입을 지정해주지만 **input을 사용하는 경우** 이야기가 달라집니다.  

어떤 데이터가 들어올 지 알기 전에 **str타입으로 미리 공간을 만들기** 때문입니다.  

간단하게 커멘드 라인에서 확인 해보면 다음과 같은 결과를 확인할 수 있습니다.  

```python
>>> birth = input("태어난 년도:")
>>> print(type(birth))
<class 'str'>
```

따라서 데이터를 입력받을 때, str이 아닌 다른 타입으로 입력을 받고자 하는 경우 미리 type casting을 해주는 것이 좋습니다.

```python
birth = int(input("태어난 년도:"))
print("나이:", 2019-birth)
```

### 2. SyntaxError: EOL while scanning string literal  

이 오류는 한 시간 동안 여러 번 발생한 오류였습니다.  

이는 문자열이 닫히지 않아 발생하는 오류입니다.  

수업 필기를 하다보면 구문주석(""")을 달게 되는 일이 많죠.  

이때, 실수로 (")를 4번 입력한 경우가 대부분이었습니다.  

혹시 수업을 듣는 도중에 이러한 오류가 발생했다면 다시 한 번 확인해보시면 원인을 찾을 수 있을 것 같습니다. 

<br>

### 3. ""와 ()의 차이를 잘 모르겠습니다.

문자열을 입력받는 ""와 함수를 실행하는 ()의 차이가 명확히 구분되지 않는다는 질문이 들어왔습니다.

말 그대로 ""는 문자열을 입력받는다는 의미입니다.  
굳이 따지자면 그림판에서 텍스트 툴을 이용해 입력받은 내용입니다.

그리고 ()는 함수나 메서드를 실행하겠다는 표시입니다.  
안에 내용이 들어갈 수도 비어있을 수도 있지만, 실행을 위해 반드시 적어주어야하지요.  
굳이 따지자면 그림판에서 텍스트를 입력받기 위해 버튼을 누르는 것과 같습니다.
